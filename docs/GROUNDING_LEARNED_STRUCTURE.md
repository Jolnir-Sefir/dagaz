# Semantic Grounding of Learned Structure — Design Document

## The Problem

The metabolic economy treats all atoms equally: passive-models, latent
variables, promoted observables, chunked actions — all pay rent, all can
die. But the semantic grounding system only knows about *declared*
observables and actions. When structure learning invents a latent variable
$L_1 from co-occurring smoke, heat, and light, that variable can predict
but cannot be explained, reasoned about compositionally, or communicated.

This is an internal inconsistency: the metabolic system says $L_1 is a
first-class citizen, but the grounding system treats it as invisible.

The `find-ungrounded` diagnostic in `action_grounding.metta` can detect
the gap but has no mechanism to close it.

## Design Principles

- **Bottom-Up**: Grounding hypotheses are generated by triggers, selected
  by metabolic pressure. No enumeration of what concepts "should" mean.
- **Uniform Schema**: Hypothesized groundings use the *same* atom schemas
  as declared groundings (`obs-grounding`, `action-grounding`, `obs-semantic`).
  The existing traversal machinery works unchanged on learned atoms.
- **Multiple Generators**: Four hypothesis generators, one selection
  mechanism. Parallels the Peircean triad: multiple ways to propose,
  one way to select.
- **Metabolic Selection**: Grounding hypotheses that enable successful
  prediction (especially communication) earn energy. Wrong groundings die.
  Same invariant as everything else.
- **Conversational Escape Hatch**: When internal generators fail, the
  system can ask its conversational partner. This emerges from EFE —
  ungrounded atoms create info-gain pressure toward clarification.

## Atom Schemas

### Grounding Hypothesis Metadata

```metta
; Tracks which groundings are hypothesized (vs declared)
; The grounding atoms themselves use standard schemas.
(grounding-hyp $target $generator $created-at)

; Metabolic energy for grounding hypotheses
(grounding-energy $target $energy)

; Prediction/reward tracking
(grounding-stats $target $tests $successes)
```

`$target` is the atom being grounded (latent variable name, promoted
observable name, or chunked action name).

`$generator` is one of: `inherited`, `behavioral`, `compositional`,
`conversational`.

### Grounding Atoms (standard schemas, same as declared)

```metta
; These are identical to what semantic_grounding.metta declares:
(obs-grounding $learned-atom $rel $concept-or-primitive)
(obs-semantic $learned-atom $description-string)

; For chunked actions:
(action-grounding $chunk-name $rel $concept-or-primitive)
(action-semantic $chunk-name $description-string)
```

The key insight: by using the same schemas, the entire existing
grounding chain traversal works on learned atoms with zero modification.
`ground-term`, `bind-observable`, `what-is-observable`, `ground-action` —
all pattern-match on these schemas and will find learned groundings
alongside declared ones.

## The Four Generators

### Generator 1: Inherited Grounding

**Trigger**: Phase 2 creates a latent variable, or Phase 3 promotes an
observable from a latent.

**Algorithm**: For each latent variable, collect all `obs-grounding`
relations of its member observables. For each `(relation, target)` pair,
count how many members share it. If the fraction exceeds the inheritance
threshold (default 0.5 — at least half the members agree), emit that
grounding for the latent.

**Example**:
```
Members of $L_1: smoke, heat, light
smoke grounds: (is-a physical-process), (causes perception)
heat  grounds: (is-a physical-process), (is-a quantity), (belongs-to environment)
light grounds: (is-a physical-process), (causes perception)
```
Intersection at threshold 0.5 (2/3 agree):
- `(obs-grounding $L_1 is-a physical-process)` — 3/3 ✓
- `(obs-grounding $L_1 causes perception)` — 2/3 ✓

Not inherited:
- `(is-a quantity)` — 1/3 ✗
- `(belongs-to environment)` — 1/3 ✗

**Strengths**: Fast, principled, handles homogeneous clusters well.

**Weaknesses**: Empty intersection for heterogeneous clusters. Cannot
capture emergent properties (fire destroys — neither smoke nor heat
does individually).

### Generator 2: Behavioral Grounding

**Trigger**: A learned atom (latent variable or promoted observable) has
passive-model connections to grounded observables, and those connections
have survived metabolic selection (energy > threshold).

**Algorithm**: For each passive-model where the learned atom is the
*cause*, check the *effect*'s existing grounding. The causal type
determines the grounding relation:

| Causal Type | Effect Grounding | Hypothesized Grounding |
|-------------|-----------------|----------------------|
| excitatory | `(obs-grounding effect is-a X)` | `(obs-grounding learned causes X)` |
| inhibitory | `(obs-grounding effect is-a X)` | `(obs-grounding learned prevents X)` |
| excitatory | `(obs-grounding effect belongs-to Y)` | `(obs-grounding learned associated Y)` |

Additionally, if the learned atom's activity (being in a high-belief
state) correlates with viability pressure, ground it with the viability
semantics:

| Behavioral Signal | Grounding |
|-------------------|-----------|
| $L_1 active → viability pressure increases | `(obs-grounding $L_1 causes danger)` |
| $L_1 active → retreat becomes EFE-optimal | `(obs-grounding $L_1 is-a threat)` |

**Example**: $L_1 has excitatory passive-models to threat-level and
ambient-temperature. Threat-level is grounded as `(is-a danger)`.
Temperature is grounded as `(is-a physical-process)`. So $L_1 gets:
- `(obs-grounding $L_1 causes danger)` — from excitatory→threat
- `(obs-grounding $L_1 causes physical-process)` — from excitatory→temp

**Strengths**: Works for heterogeneous clusters. Captures emergent
causal properties. Grounds by what the concept *does*, not what it *is*.

**Weaknesses**: Requires established passive-models with metabolic
energy — takes several cycles to generate. May produce multiple
competing groundings.

### Generator 3: Compositional Grounding

**Trigger**: Phase 4 reifies a chunked action from a repeated policy.

**Algorithm**: Compose the groundings of the component actions in
sequence. The chunk's grounding is a structured composition:

```metta
(action-grounding observe-retreat is-a (sequence observe retreat))
(action-grounding observe-retreat causes (know then (away threat)))
(action-grounding observe-retreat enables (informed-action))
(action-semantic observe-retreat "gather information then move to safety")
```

The composition rules:
1. The chunk `is-a` a `sequence` of its components.
2. The chunk `causes` the sequential composition of component effects.
3. The chunk `enables` anything that the *last* action enables
   (the strategic outcome).
4. The chunk `requires` anything that the *first* action requires
   (the entry condition).
5. The semantic description concatenates component descriptions
   with temporal connectives.

**Strengths**: Principled, predictable, preserves compositional structure.

**Weaknesses**: Cannot capture genuinely emergent strategic meaning
("cautious retreat" implies a *disposition*, not just a sequence).
For that, behavioral grounding of the chunk's action-model performance
would supplement.

### Generator 4: Conversational Acquisition

**Trigger**: The system detects an ungrounded active atom (via
`find-ungrounded` extended to cover learned atoms) AND a conversation
is active.

**Mechanism**: This generator doesn't directly create grounding atoms
in MeTTa. Instead, it:

1. Creates a `(grounding-request $atom $members $behavioral-sig)` atom
   that signals the orchestrator.
2. The orchestrator reads this, constructs an LLM prompt:
   "I observe that [member descriptions] always co-occur. What single
   concept describes this pattern?"
3. The LLM response is parsed into a label and semantic type.
4. The orchestrator injects:
   ```metta
   (obs-grounding $atom is-a $parsed-type)
   (obs-semantic $atom $parsed-label)
   (grounding-hyp $atom conversational $cycle)
   (grounding-energy $atom 0.5)
   ```

**EFE Integration**: The ungrounded atom creates a low-precision belief
on `grounding-completeness` (self-model observable), which creates
info-gain for conversational actions. The system should spontaneously
ask about concepts it can't explain — the "Sherlock Holmes effect"
applied to self-understanding.

**Strengths**: Catches everything the internal generators miss. Grounds
concepts in the conversational partner's ontology. The only generator
that can produce a *name* for a learned concept.

**Weaknesses**: Depends on conversation being active. LLM-generated
groundings are low-precision (treated as uncertain hypotheses).
Introduces a dependency on the orchestrator.

## Metabolic Management

### Energy Lifecycle

```
Create (energy = grounding-initial-energy, default 0.6)
  → Gestation (grace period, no rent)
  → Active (pay rent each cycle, earn energy from successful use)
  → Death (energy ≤ 0 → remove grounding atoms)
```

### Reward Signal

Grounding hypotheses earn energy when they contribute to successful
outcomes. There are three reward channels:

1. **Communication reward**: The system uses a grounded learned concept
   in verbalization (via `what-am-i-doing` or similar), and the partner's
   comprehension observable doesn't spike downward. The grounding helped
   communicate. This is the primary reward.

2. **Reasoning reward**: The grounded concept participates in a
   deductive chain that crosses the grounded/ungrounded boundary, and
   the deduced link earns metabolic energy. The grounding enabled
   useful reasoning.

3. **Self-model reward**: The `grounding-completeness` observable moves
   closer to its preference when a grounding is created, reducing
   prediction error on the self-model. Small but consistent reward for
   closing the gap.

### Multiple Competing Groundings

It's possible (expected) that multiple generators produce different
groundings for the same atom. This is fine — they compete metabolically.
The inherited grounding says $L_1 `is-a physical-process`. The
behavioral grounding says $L_1 `causes danger`. The conversational
grounding says $L_1 `is-a fire`. All three can coexist and all three
can be used in different contexts. The ones that enable successful
prediction and communication survive. No need to choose one.

This is analogous to how multiple passive-models can predict the same
observable. The metabolic system handles competition without requiring
an explicit arbitration mechanism.

## Configuration

```metta
; Grounding hypothesis generation
(config grounding-inheritance-threshold 0.5)  ; Fraction of members that must agree
(config grounding-behavioral-min-energy 0.5)  ; Passive model must be this stable
(config grounding-initial-energy 0.6)         ; Starting energy for hypotheses
(config grounding-metabolic-rate 0.02)        ; Rent per cycle (lower than structure)
(config grounding-metabolic-boost 0.08)       ; Reward for successful use
(config grounding-gestation-period 3)         ; Grace period before rent
(config grounding-max-per-cycle 3)            ; Budget cap on new hypotheses
```

The metabolic rate for groundings is lower than for causal structure
(0.02 vs 0.05) because groundings are tested less frequently — they
only earn reward when the system explains itself or communicates, not
every prediction cycle.

## Cycle Integration

The grounding step runs AFTER structure learning and BEFORE action
selection in `cognitive-cycle-v2!`:

```
... structure learning step ...
... lifecycle metabolic step ...
→ GROUNDING STEP (new):
    1. Check for newly created learned atoms (latent vars, promoted obs, chunks)
    2. Run Generator 1 (inherited) on new latent variables
    3. Run Generator 2 (behavioral) on all ungrounded learned atoms with
       established passive models
    4. Run Generator 3 (compositional) on new chunked actions
    5. Signal Generator 4 (conversational acquisition) for remaining gaps
    6. Drain grounding energy (metabolic rent)
    7. Prune dead groundings
    8. Update grounding-completeness self-observable
→ action selection (which may now use grounded learned concepts in EFE trace) ...
```

The grounding step is a REFLEX — it fires when conditions are met,
not selected by EFE. This parallels structure learning.

## Self-Model Observable

```metta
(belief grounding-completeness $ratio $precision)
(preference grounding-completeness 1.0 0.6)
```

Updated each cycle: ratio = (grounded active atoms) / (total active atoms).
Precision starts low (0.3) and increases as the system stabilizes.

This creates continuous EFE pressure toward self-understanding:
- Low grounding-completeness → high prediction error on self-model
- Conversational actions that improve grounding → reduce this error
- The system *wants* to understand its own concepts

## Interaction with Existing Modules

### semantic_grounding.metta
No changes needed. The `what-is-observable`, `describe-observable`,
`grounded-in?`, and `is-grounded?` functions all pattern-match on
`obs-grounding` atoms. Hypothesized groundings use the same schema,
so they're found automatically.

### action_grounding.metta
No changes needed. `ground-action`, `ground-term`, `term-status` all
work on the standard schemas. `find-ungrounded` naturally finds
ungrounded learned atoms because they lack `obs-grounding` atoms
until a generator creates them.

### self_model.metta
Add `grounding-completeness` as a self-observable with queries:
- `(grounding-coverage)` — ratio of grounded to total learned atoms
- `(ungrounded-atoms)` — list of active atoms without grounding
- `(grounding-sources $atom)` — which generator(s) grounded this atom

### conversation_model.metta
No structural changes. The grounding-request signal for Generator 4
is read by the orchestrator, not by the conversation model. The
conversation model's existing EFE machinery naturally selects
clarification actions when grounding-completeness is low.

## Validation Strategy

### Unit Tests (Python benchmark)

1. **Inheritance test**: Create latent variable with known member
   groundings, verify intersection computed correctly.
2. **Behavioral test**: Create passive-models from learned atom to
   grounded observables, verify causal grounding generated.
3. **Compositional test**: Create chunked action from known components,
   verify sequential composition of groundings.
4. **Metabolic death test**: Create wrong grounding, verify it dies
   within ~30 cycles when it doesn't earn reward.
5. **Competing groundings test**: Create multiple groundings for same
   atom, verify all coexist and best one earns most energy.

### Integration Tests

6. **Sherlock Holmes for grounding**: Seed an ungrounded latent variable,
   verify that grounding-completeness drops, info-gain for clarification
   rises, and the system selects a conversational action.
7. **End-to-end narrative**: Create a latent variable, let inherited
   grounding fire, verify `what-am-i-doing` can now describe the
   concept in its narrative output.

## Known Limitations

1. **No metaphorical grounding**: All generators produce literal
   semantic relations. A latent variable that functions *like* fear
   but isn't literally fear has no mechanism to be grounded as "similar
   to fear." This would require an analogy mechanism.

2. **Grounding depth is shallow**: Behavioral grounding looks one hop
   from the learned atom to grounded observables. Deep causal chains
   (learned → learned → grounded) are not followed. This parallels
   the deductive reasoning limit of chain length 2.

3. **No negative grounding**: The generators can hypothesize what a
   concept IS but not what it ISN'T. Contrastive grounding ("$L_1 is
   like heat but not like sound") would require comparing against
   rejected hypotheses.

4. **Conversational acquisition assumes cooperative partner**: If the
   partner gives wrong labels, the grounding is wrong. Metabolic
   selection should eventually kill it (the label won't help predict),
   but the system has no mechanism to detect deliberate misinformation
   about concept semantics.
